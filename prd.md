# 📝 PRD: 독서 기록 달력 웹앱 (BookLog)

## 1. 프로젝트 개요 (Project Overview)
* **목적:** 사용자가 매일 읽은 책의 페이지나 시간을 달력 형태로 기록하여 독서 습관을 시각화하고 동기를 부여함.
* **타겟 사용자:** 꾸준한 독서를 원하는 사람, 내가 읽은 책의 기록을 체계적으로 남기고 싶은 독서가.
* **핵심 가치:** "기록의 시각화"와 "독서 데이터의 자산화".

---

## 2. 주요 기능 (Key Features)

### 2.1. 대시보드 & 달력 뷰 (Main Calendar)
* **월간 달력:** 날짜별로 읽은 책의 표지 이미지나 진행률 표시.
* **독서 상태 요약:** 이번 달 읽은 권수, 총 페이지 수, 목표 달성률 상단 표시.
* **미니 통계:** 주간/월간 독서량 추이 그래프.

### 2.2. 독서 로그 기록 (Logging)
* **책 검색:** 외부 도서 API(예: 알라딘, 카카오 도서 검색)를 통해 도서 정보(제목, 저자, 표지, 전체 페이지) 자동 입력.
* **데일리 기록:** 읽은 페이지 범위 입력, 짧은 메모(한 줄 평), 독서 시간 기록.
* **상태 관리:** '읽고 싶은', '읽는 중', '완독' 상태 구분.

### 2.3. 독서 분석 (Analytics)
* **장르 분포:** 내가 주로 읽는 카테고리 비율(파이 차트).
* **독서 패턴:** 가장 많이 읽는 요일이나 시간대 분석.

### 2.4. 독서 기록 공유 (SNS Share)
* **공유용 이미지 생성:** 특정 월의 독서 달력과 요약 통계(완독 권수, 총 페이지, 연속 독서일)를 깔끔한 카드 형태로 시각화.
* **SNS 최적화 디자인:** 인스타그램 등 SNS 피드에 최적화된 4:5 비율 디자인과 감성적인 테마(완독 스티커 등) 제공.

---

## 3. 사용자 흐름 (User Flow)



1.  **로그인/가입:** 소셜 로그인(구글, 카카오 등)을 통해 간편하게 진입.
2.  **책 추가:** 검색 창에서 읽기 시작한 책을 선택하여 서재에 등록.
3.  **기록하기:** 달력에서 오늘 날짜를 클릭하고 읽은 페이지를 입력.
4.  **확인:** 달력에 책 표지가 채워지는 것을 보며 성취감 확인.

---

## 4. 기술 스택 제안 (Technical Stack)

| 구분 | 추천 기술 | 이유 |
| :--- | :--- | :--- |
| **Frontend** | React.js / Next.js | 컴포넌트 기반으로 달력 UI 구현이 용이함. |
| **Styling** | Tailwind CSS | 빠르고 반응형인 UI 디자인 가능. |
| **Database** | Firebase / Supabase | 별도의 백엔드 구축 없이 실시간 DB와 인증 해결. |
| **Library** | FullCalendar / Recharts | 달력 로직 및 통계 시각화 라이브러리 활용. |

---

## 5. 상세 기능 요구사항 (Detailed Requirements)

### **R1. 도서 관리**
* 사용자는 제목이나 ISBN으로 책을 검색할 수 있어야 한다.
* 직접 표지 이미지를 업로드하거나 정보를 수정할 수 있어야 한다.

### **R2. 달력 커스터마이징**
* 완독한 날에는 달력 해당 칸에 '완독' 스티커나 특별한 하이라이트 표시.
* 연속 독서 일수(Streak)를 표시하여 재미 요소 추가.

### **R3. 반응형 디자인**
* 모바일에서도 편하게 기록할 수 있도록 웹 최적화(PWA 고려).

### **R4. 이미지 저장 및 공유**
* 사용자가 '공유하기' 기능을 통해 선택한 월의 달력과 독서 통계를 이미지로 변환(캡처)하여 다운로드하거나 SNS에 바로 공유할 수 있어야 한다.
