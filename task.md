# 독서 기록 달력 웹앱 (BookLog) - 작업 현황 정리

## 1. 프로젝트 초기화 및 환경 설정
- **프레임워크**: Vite + React 기반
- **스타일링**: Tailwind CSS v4 적용
- **레이아웃 디자인**: 세련된 '글래스모피즘(Glassmorphism)'과 `primary(Indigo)` 및 다채로운 그라디언트를 사용한 프리미엄 UI 구축

## 2. 주요 아키텍처 및 상태 관리
- **전역 상태 관리**: `Jotai` 도입 (`src/store.js`)
  - `readingsAtom`: 독서 기록 원본 데이터 관리
  - `addReadingAtom`: 새로운 독서 기록을 전역 상태에 추가하는 액션
  - `statsAtom`: 전체 기록을 기반으로 달성률, 월간 누적 독서량 등 필요한 파생 데이터(Derived State) 실시간 자동 계산
  - `currentViewAtom`: 사이드바(GNB)를 통한 화면 라우팅(Calendar, Library, Stats) 제어
- **날짜 라이브러리 교체**: `date-fns`를 제거하고 더 가벼운 `dayjs`로 달력 렌더링 및 모달 날짜 포맷 로직 전면 마이그레이션 적용

## 3. 핵심 기능 구현 내용

### 📆 1) 달력 대시보드 (`CalendarWidget`, `StatsWidget`)
- `dayjs`를 활용하여 이번 달의 주차/일자를 동적으로 계산해 나열하고 스타일링
- 해당 일자에 읽은 책이 겹치면 표지 이미지(Cover UI)와 진행률 바(Progress bar)를 삽입
- '완독' 뱃지 및 마우스 오버 시 간략 정보를 보여주는 말풍선(Tooltip) 표시
- 상단에서는 Jotai 상태를 가져와 '이번 달 완독', '총 읽은 페이지', '달성률' 등의 통계를 수려한 컴포넌트로 보여줌

### ✍️ 2) 기록하기 모달 (`RecordModal`)
- **[핵심기능] 도서 API 연동**:
  - 기존 구글 도서 API에서 **국립중앙도서관 OpenAPI**로 전면 교체
  - 로컬 개발 환경 CORS 우회를 위해 `vite.config.js`에 `/nl-api` 프록시를 설정해 안정적 호출(`https://www.nl.go.kr`) 보장
  - XML 기반 응답 결과를 브라우저 내장 `DOMParser`로 직접 파싱하여 JSON 형태로 정제 후 드롭다운 목록에 제공 (제목, 저자, 썸네일 등)
- 데이터 입력 폼: 시작/끝 페이지, 독서 시간, 한 줄 평, 읽는 상태('읽는 중'/'완독') 등 관리

### 📚 3) 내 서재 (`LibraryView`)
- 달력을 넘어, 지금까지 읽거나 완독한 책 목록만 한눈에 모아보는 뷰
- '전체 / 읽는 중 / 완독' 상태 탭을 둬서 필터링 기능 제공
- 표지를 메인으로 하는 그리드형 갤러리로 구성 (마이너한 마우스 호버 이펙트로 한 줄 평 엿보기 제공)

### 📊 4) 통계 인사이트 (`StatsView`)
- `Recharts` 라이브러리를 사용하여 시각화 강화
- **원형 차트(Pie Chart)**: (현재 Mock 데이터 기반) 선호하는 취향 장르의 비율을 보여줌
- **막대 차트(Bar Chart)**: 월별 누적 완독 권수를 나타내어 독서량의 추세를 한눈에 확인 가능

## 4. 반응형(Responsive) UI 최적화 완료
- **전역 레이아웃 및 사이드바 (`App.jsx`, `Sidebar.jsx`)**:
  - 레이아웃 전체에 수평 스크롤(오버플로우)이 발생하지 않도록 루트 컨테이너(`overflow-x-hidden`, `min-w-0`) 제약 로직 추가.
  - 모바일(768px 미만)에서는 하단 고정 네비게이션 바(Bottom Navigation Bar) 형태로 동작하고, 데스크탑에서는 좌측 사이드 패널로 작동하도록 동적 레이아웃 구성.
- **달력 대시보드 (`CalendarWidget.jsx`)**:
  - **모바일 최적화**: 좁은 화면을 넓게 쓰도록 날짜 사이의 간격 제거(`gap-0`), 네모반듯한 테이블 형태 구현을 위해 둥근 테두리 축소(`rounded-none`).
  - **렌더링 버그 수정**: 오늘 날짜를 강조하는 링(`ring`) 아웃라인이 다른 날짜 패널에 가려지지 않도록 쌓임 순서(`z-10`) 상승 부여.
  - **높이 확장 보장**: 달력 내 총 주차가 6주 이상 렌더링될 때 배경 박스 하단이 뚫리는 버그를 막기 위해 특정 높이(`md:h-[700px]`) 강제를 해제하고 `h-auto` 속성으로 콘텐츠 길이에 자연스럽게 늘어나도록 조치.
- **기타 뷰 및 모달 최적화 (`RecordModal`, `LibraryView`, `StatsView`)**:
  - 모바일 기기에서도 입력 폼, 차트, 리스트, 필터 옵션들이 자연스럽게 줄바꿈 및 전체 가로폭(`1열 grid`, 세로 나열)을 가지도록 보완.

## 5. 도서 검색 API 고도화 (국립중앙도서관 Open API)
- **API 응답 포맷 전환 (XML ➔ JSON)**:
  - 자원 소모가 많은 DOM 구조 탐색 로직(DOMParser)을 걷어내고, 응답 파라미터(`apiType=json`)를 추가해 직관적인 JSON 객체 기반의 데이터 파싱 방식으로 코드 최적화.
  - JSON 형식에 맞추어 `titleInfo`, `authorInfo`, `imageUrl` 등 카멜 케이스(CamelCase) 키 값을 매핑.
- **상세검색 조건(`detailSearch`) 추가**:
  - 명세서(`api_spec.md`)의 `f1`, `v1` 쿼리 파생 규칙을 반영하여, 사용자가 **전체, 제목, 저자, ISBN**으로 타깃을 좁혀 검색할 수 있는 탭 UI와 조건부 파라미터 핸들러(`URLSearchParams`) 구축.
- **페이징(Pagination) 컨트롤 UI 도입**:
  - 수천 건의 데이터도 안전하게 볼 수 있도록 `total` 속성 기반 최대 페이지 수(총량 ÷ 페이지 사이즈) 계산 및 [이전/다음] 버튼 연동.
- **문자열 파싱 강화 (HTML 태그 제거)**:
  - JSON 응답 값의 문자열(예: `<span class="searching_txt">토지</span>`)이 화면에 노출되는 버그를 고치기 위해, 정규식(`/ <[^>]*>?/gm `)을 사용해 순수 텍스트만 추출하도록 렌더링 품질 개선.
