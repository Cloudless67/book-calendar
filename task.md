# 독서 기록 달력 웹앱 (BookLog) - 작업 현황 정리

## 1. 프로젝트 초기화 및 환경 설정
- **프레임워크**: Vite + React 기반
- **스타일링**: Tailwind CSS v4 적용
- **레이아웃 디자인**: 세련된 '글래스모피즘(Glassmorphism)'과 `primary(Indigo)` 및 다채로운 그라디언트를 사용한 프리미엄 UI 구축

## 2. 주요 아키텍처 및 상태 관리
- **전역 상태 관리**: `Jotai` 도입 (`src/store.js`)
  - `readingsAtom`: 독서 기록 원본 데이터 관리
  - `addReadingAtom`: 새로운 독서 기록을 전역 상태에 추가하는 액션
  - `statsAtom`: 전체 기록을 기반으로 달성률, 월간 누적 독서량 등 필요한 파생 데이터(Derived State) 실시간 자동 계산
  - `currentViewAtom`: 사이드바(GNB)를 통한 화면 라우팅(Calendar, Library, Stats) 제어
- **날짜 라이브러리 교체**: `date-fns`를 제거하고 더 가벼운 `dayjs`로 달력 렌더링 및 모달 날짜 포맷 로직 전면 마이그레이션 적용

## 3. 핵심 기능 구현 내용

### 📆 1) 달력 대시보드 (`CalendarWidget`, `StatsWidget`)
- `dayjs`를 활용하여 이번 달의 주차/일자를 동적으로 계산해 나열하고 스타일링
- 해당 일자에 읽은 책이 겹치면 표지 이미지(Cover UI)와 진행률 바(Progress bar)를 삽입
- '완독' 뱃지 및 마우스 오버 시 간략 정보를 보여주는 말풍선(Tooltip) 표시
- 상단에서는 Jotai 상태를 가져와 '이번 달 완독', '총 읽은 페이지', '달성률' 등의 통계를 수려한 컴포넌트로 보여줌

### ✍️ 2) 기록하기 모달 (`RecordModal`)
- **[핵심기능] 도서 API 연동**:
  - 기존 구글 도서 API에서 **국립중앙도서관 OpenAPI**로 전면 교체
  - 로컬 개발 환경 CORS 우회를 위해 `vite.config.js`에 `/nl-api` 프록시를 설정해 안정적 호출(`https://www.nl.go.kr`) 보장
  - XML 기반 응답 결과를 브라우저 내장 `DOMParser`로 직접 파싱하여 JSON 형태로 정제 후 드롭다운 목록에 제공 (제목, 저자, 썸네일 등)
- 데이터 입력 폼: 시작/끝 페이지, 독서 시간, 한 줄 평, 읽는 상태('읽는 중'/'완독') 등 관리

### 📚 3) 내 서재 (`LibraryView`)
- 달력을 넘어, 지금까지 읽거나 완독한 책 목록만 한눈에 모아보는 뷰
- '전체 / 읽는 중 / 완독' 상태 탭을 둬서 필터링 기능 제공
- 표지를 메인으로 하는 그리드형 갤러리로 구성 (마이너한 마우스 호버 이펙트로 한 줄 평 엿보기 제공)

### 📊 4) 통계 인사이트 (`StatsView`)
- `Recharts` 라이브러리를 사용하여 시각화 강화
- **원형 차트(Pie Chart)**: (현재 Mock 데이터 기반) 선호하는 취향 장르의 비율을 보여줌
- **막대 차트(Bar Chart)**: 월별 누적 완독 권수를 나타내어 독서량의 추세를 한눈에 확인 가능

## 4. 반응형(Responsive) UI 최적화 완료
- **전역 레이아웃 및 사이드바 (`App.jsx`, `Sidebar.jsx`)**:
  - 레이아웃 전체에 수평 스크롤(오버플로우)이 발생하지 않도록 루트 컨테이너(`overflow-x-hidden`, `min-w-0`) 제약 로직 추가.
  - 모바일(768px 미만)에서는 하단 고정 네비게이션 바(Bottom Navigation Bar) 형태로 동작하고, 데스크탑에서는 좌측 사이드 패널로 작동하도록 동적 레이아웃 구성.
- **달력 대시보드 (`CalendarWidget.jsx`)**:
  - **모바일 최적화**: 좁은 화면을 넓게 쓰도록 날짜 사이의 간격 제거(`gap-0`), 네모반듯한 테이블 형태 구현을 위해 둥근 테두리 축소(`rounded-none`).
  - **렌더링 버그 수정**: 오늘 날짜를 강조하는 링(`ring`) 아웃라인이 다른 날짜 패널에 가려지지 않도록 쌓임 순서(`z-10`) 상승 부여.
  - **높이 확장 보장**: 달력 내 총 주차가 6주 이상 렌더링될 때 배경 박스 하단이 뚫리는 버그를 막기 위해 특정 높이(`md:h-[700px]`) 강제를 해제하고 `h-auto` 속성으로 콘텐츠 길이에 자연스럽게 늘어나도록 조치.
- **기타 뷰 및 모달 최적화 (`RecordModal`, `LibraryView`, `StatsView`)**:
  - 모바일 기기에서도 입력 폼, 차트, 리스트, 필터 옵션들이 자연스럽게 줄바꿈 및 전체 가로폭(`1열 grid`, 세로 나열)을 가지도록 보완.

## 5. 도서 검색 API 고도화 (국립중앙도서관 Open API)
- **API 응답 포맷 전환 (XML ➔ JSON)**:
  - 자원 소모가 많은 DOM 구조 탐색 로직(DOMParser)을 걷어내고, 응답 파라미터(`apiType=json`)를 추가해 직관적인 JSON 객체 기반의 데이터 파싱 방식으로 코드 최적화.
  - JSON 형식에 맞추어 `titleInfo`, `authorInfo`, `imageUrl` 등 카멜 케이스(CamelCase) 키 값을 매핑.
- **상세검색 조건(`detailSearch`) 추가**:
  - 명세서(`api_spec.md`)의 `f1`, `v1` 쿼리 파생 규칙을 반영하여, 사용자가 **전체, 제목, 저자, ISBN**으로 타깃을 좁혀 검색할 수 있는 탭 UI와 조건부 파라미터 핸들러(`URLSearchParams`) 구축.
- **페이징(Pagination) 컨트롤 UI 도입**:
  - 수천 건의 데이터도 안전하게 볼 수 있도록 `total` 속성 기반 최대 페이지 수(총량 ÷ 페이지 사이즈) 계산 및 [이전/다음] 버튼 연동.
- **문자열 파싱 강화 (HTML 태그 제거)**:
  - JSON 응답 값의 문자열(예: `<span class="searching_txt">토지</span>`)이 화면에 노출되는 버그를 고치기 위해, 정규식(`/ <[^>]*>?/gm `)을 사용해 순수 텍스트만 추출하도록 렌더링 품질 개선.

## 6. 백엔드 연동 및 전역 상태(Store) 고도화
- **Supabase 연동**: 기존 MockData(하드코딩) 기반에서 Supabase 데이터베이스 기반으로 마이그레이션 적용.
  - `loadReadingsAtom`, `addReadingAtom`, `updateReadingAtom`, `deleteReadingAtom` 구현.
  - 환경변수(`.env`)로 `VITE_SUPABASE_URL` 및 `VITE_SUPABASE_ANON_KEY` 분리.
- **다중 데이터(배열) 벌크 인서트**: 여러 날 기록 시 Supabase에 `insert`할 때 단일 객체 뿐만 아니라 배열을 한 번에 전송 가능하도록 로직 정비.

## 7. 도서 검색 API 전면 교체 (Naver + 서지 하이브리드)
- **네이버 도서 검색 API 도입**: 기존 국립중앙도서관 API의 썸네일/표지 이미지 미제공 한계를 해결하기 위해 네이버 API로 교체.
- **쪽수(Page) 데이터 보완**: 네이버 API가 제공하지 않는 '총 페이지 수'는 기존 서지(Seoji) OpenAPI를 ISBN 기반으로 백그라운드에서 병행 호출하여 두 API의 장점을 모두 취함.
- **CORS 프록시 설정**: `vite.config.js`에 `/naver-api` 프록시를 추가하여 네이버 서버 요청 보안 이슈 우회.

## 8. UX/UI 개선 및 신규 기능 (서재 & 달력)
- **서재 상세 모달(`LibraryDetailModal`)**:
  - 내 서재에서 특정 책을 클릭 시 상세 모달 오픈.
  - 메모, **인용구(Quotes)** 관리 기능 추가 및 DB 연동.
  - 별점(⭐) 부여 기능(1~5점)을 인터랙티브하게 구현.

- **기록 모달(`RecordModal`) 다중 날짜(Multi-day) 기능**:
  - 한 권의 책을 여러 날에 걸쳐 읽었을 때 기록을 한 번에 배분하는 기능 추가.
  - 거대한 버튼 대신 세련된 **iOS 스타일 토글 스위치(Toggle Switch)**와 애니메이션 날짜 인풋창을 구현하여 완성도 향상.
  - 선택한 날짜 수에 맞춰 읽은 페이지(pagesRead)와 독서 시간(readingTime)이 N분의 1로 자동 균등 분할 저장되도록 알고리즘 구현.
- **캘린더 드래그 앤 드롭(Drag & Drop) 선택**:
  - `CalendarWidget`에서 캘린더 일자를 마우스로 누른 채 드래그하여 여러 날을 선택할 수 있는 UX 연동.
  - 역방향 드래그 자동 정렬, 드래그 영역의 시각적 하이라이트(보라색) 처리.
  - 마우스를 뗄 시 선택된 기간이 자동 세팅된 채 모달이 열림.

## 10. 캘린더 모바일 인터페이스 고도화 및 버그 수정
- **빈 날짜 컴포넌트 시각화(Empty State)**: 
  - 딱딱한 기본 `+` 버튼 대신 점선 형태의 '빈 책 표지' UI를 구현하여 사용자가 채우고 싶은 감성적 디자인 제공 및 복잡도(클러터링) 최소화.
- **다중 도서 팝오버(Popover) 시야 최적화**: 
  - 달력의 특정 요일(예: 일, 월 혹은 금, 토)에 팝오버를 열 때 화면 좌우 밖으로 잘리는 버그를 고치기 위해, 요일 기반 조건부 렌더링으로 팝오버 전개 방향을 다이내믹하게 조정(`left-0`, `right-0`).
  - 달력 최하단 2줄에서 팝오버를 열면 화면 밑으로 잘리던 문제를 해결하기 위해, 마지막 행 여부를 계산하여 위로 솟아오르는(`bottom-full`) Pop-up 렌더링 지원.
  - 마우스 호버 시 발생하는 말풍선이나 팝오버가 달력 내부의 다른 뱃지(`+`)나 날짜 텍스트 등에 가려지지 않도록 컨테이너 간 세밀한 `z-index` 스태킹 컨텍스트 재조정.
- **모달 레이아웃 및 제어 최적화 (`RecordModal`, `ShareModal`)**:
  - 모달 팝업 시 뒷배경(본문) 스크롤이 이중으로 동작하는 것을 막기 위해 `overflow: hidden` 처리 완비.
  - 디바이스 세로 길이가 짧은 모바일 환경에서, 모달 높이가 길어 저장 버튼이 잘리는 현상을 막기 위해 Tailwind UI 공식 패턴(`flex min-h-full`)과 래퍼(`overflow-y-auto`)를 사용해 안전한 자체 스크롤 보장.

## 11. API 및 썸네일 캐싱 시스템 구축
- **프론트엔드 캐싱 (React Query 연동)**:
  - `@tanstack/react-query`를 도입하여 네이버 도서 검색 API의 응답을 브라우저 메모리에 캐싱.
  - 사용자가 동일한 검색어를 재입력할 때 API를 다시 찌르지 않고 즉각적으로 결과를 렌더링하도록 성능 최적화.
- **백엔드 영구 캐싱 (Supabase 연동)**:
  - 서지 API(ISBN 기반 쪽수 조회) 호출 비용을 줄이고자 Supabase에 `books` 테이블 캐시 레이어 설계(`supabase_setup.sql` 작성).
  - 도서 선택 시 DB에 캐시된 책 정보가 있다면 즉시 반환하고, 없다면 서지 API를 호출하여 DB에 자동 적재(Insert).
- **썸네일 이미지 자체 CDN 서빙 (Supabase Storage)**:
  - 네이버에서 제공하는 썸네일 URL을 직접 사용하지 않고, 프록시를 통해 Blob으로 다운로드한 뒤 내 Supabase `book-covers` 버킷에 자동 업로드하도록 아키텍처 개편.
  - 브라우저에 보여지는 이미지는 영구적으로 보존되는 Supabase 글로벌 CDN URL로 교체되어 외부 서비스 의존도를 완전히 분리.

## 12. 데이터 모델 개선 (`startPage`, `endPage` 도입)
- **절대 페이지 관리 체계로 전환**: 기존의 읽은 분량(`pagesRead`) 중심 저장 방식에서, 독서의 시작점과 끝점을 명확히 기록하는 `startPage` 및 `endPage` 컬럼 체계로 Supabase 스키마 고도화.
- **연속 독서 UX 개선**: 기록 추가 시 해당 도서의 마지막 `endPage`를 자동으로 찾아 다음 기록의 `startPage`로 제안하는 스마트 필링 로직 구현.
- **진행률 계산 정밀화**: 서재 뷰 및 캘린더 등 모든 UI의 진행률 계산을 `endPage / totalPages`의 절대 비율 방식으로 변경하여 데이터 신뢰도 향상.
- **하이브리드 호환성 유지**: `startPage/endPage`가 없는 과거 데이터에 대해서도 `pagesRead`를 활용해 끊김 없이 통계를 보여주는 폴백(Fallback) 로직 적용.

## 13. 기록 모달(`RecordModal`) 대규모 리팩토링 및 모듈화
- **파일 사이즈 대폭 슬림화**: 730라인이 넘던 단일 파일을 기능별 서브 컴포넌트로 분리하여 가독성 및 유지보수성 극대화 (약 230라인으로 축소).
- **비즈니스 로직 추출 (`src/lib/bookApi.js`)**: 네이버 검색, 국립중앙도서관 API 연동, 이미지 프록시 및 Supabase Storage 업로드 등의 복잡한 비동기 로직을 캡슐화하여 UI 코드와 완전 분리.
- **UI 섹션 컴포넌트화 (`src/components/record/`)**:
  - `DateSelector`: 날짜 및 멀티 데이 설정 UI.
  - `BookSearch`: 도서 검색 입력 및 결과 목록 UI.
  - `RecentBooks`: 최근 읽은 책 선택 UI.
  - `BookSelectedCard`: 선택된 도서 요약 정보 UI.
  - `RecordFormFields`: 페이지, 시간, 메모, 상태 입력 UI.

*추후 고도화할 목표 혹은 TODO가 있다면 본 문서의 하단에 덧붙이고 이어서 진행할 수 있습니다.*

## 14. PWA(Progressive Web App) 지원 추가
- **`vite-plugin-pwa` 도입**: 빌드 시 Web Manifest 및 Service Worker를 자동 생성하여 PWA 환경 구축.
- **아이콘 및 에셋 추가**: 기존 서비스 로고(`booklog.png`)를 활용하여 PWA 필수 아이콘(192x192, 512x512) 및 iOS용 `apple-touch-icon`(180x180) 생성 후 `public` 폴더에 적재.
- **메타 태그 및 모바일 뷰포트 최적화**: 모바일 기기에서 네이티브 앱처럼 자연스럽게 보이도록 핀치 줌 방지(`viewport`) 속성을 수정하고, 주소창 테마 색상(`theme-color`)을 `index.html`에 명시.
- **백그라운드 자동 업데이트**: Service Worker의 `registerType`을 `autoUpdate`로 지정하여 캐싱된 리소스가 갱신될 때 사용자 측에서 자동으로 업데이트를 적용받도록 구현.
